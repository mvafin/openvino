# FP8 Quantize/Dequantize ONNX Model Examples

This directory contains example ONNX models with FP8 (8-bit floating-point) Quantize/Dequantize operations, demonstrating patterns commonly output by quantization tools like NVIDIA ModelOpt and AMD Quark.

## Background

FP8 quantization is a technique for reducing model size and accelerating inference by using 8-bit floating-point formats instead of traditional FP32. Two primary FP8 formats are standardized:

1. **E4M3 (FLOAT8E4M3FN)**: 1 sign bit, 4 exponent bits, 3 mantissa bits
   - Wider dynamic range
   - Better for activations and gradients
   - Used extensively by NVIDIA (Hopper architecture and newer)

2. **E5M2 (FLOAT8E5M2)**: 1 sign bit, 5 exponent bits, 2 mantissa bits
   - Higher precision within a smaller range
   - Better for weights in some cases
   - Supported by various hardware accelerators

## Models Included

### 1. NVIDIA ModelOpt FP8 E4M3 Example (`nvidia_modelopt_fp8_e4m3.onnx`)

**Generated by**: `generate_nvidia_modelopt_fp8.py`

**Description**: Simulates the FP8 quantization pattern typically produced by NVIDIA TensorRT Model Optimizer (ModelOpt).

**Key Characteristics**:
- Uses FP8 E4M3FN format for both activations and weights
- Per-tensor quantization with dynamic scales for activations
- **Weights are pre-quantized and folded into FP8 constants** (no runtime QuantizeLinear for weights)
- Pattern: `Input -> QuantizeLinear -> DequantizeLinear -> MatMul -> Output`
- Optimized for NVIDIA GPU architectures (Hopper H100, Blackwell, etc.)

**Model Structure**:
```
Input (FP32, shape: [1, 4])
  |
  v
QuantizeLinear (scale: 0.05)
  |
  v
input_quantized (FP8 E4M3FN)
  |
  v
DequantizeLinear (scale: 0.05)
  |
  v
input_dequantized (FP32) ----+
                              |
weight_fp8 (FP8 E4M3FN) ------+---> MatMul --> Output (FP32, shape: [1, 4])
  |                           |
  v                           |
DequantizeLinear (scale: 1.0)-+
  |
  v
weight_dequantized (FP32)
```

**Note**: The weight is pre-quantized and stored directly as an FP8 E4M3FN constant, eliminating the need for runtime quantization.

**Reference**: [NVIDIA TensorRT Model Optimizer](https://github.com/NVIDIA/TensorRT-Model-Optimizer)

### 2. AMD Quark FP8 E5M2 Example (`amd_quark_fp8_e5m2.onnx`)

**Generated by**: `generate_amd_quark_fp8.py`

**Description**: Simulates the FP8 quantization pattern typically produced by AMD Quark quantization toolkit.

**Key Characteristics**:
- Uses FP8 E5M2 format for both activations and weights
- Per-channel quantization for weights (more fine-grained than per-tensor)
- Pattern: `Input -> QuantizeLinear -> DequantizeLinear -> Conv -> Output`
- Cross-platform support (AMD, NVIDIA, CPU)

**Model Structure**:
```
Input (FP32, shape: [1, 3, 8, 8])
  |
  v
QuantizeLinear (scale: 0.1) -> input_quantized (FP8 E5M2)
  |
  v
DequantizeLinear (scale: 0.1) -> input_dequantized (FP32)
  |
  v
Conv2D (with FP8 quantized weights, per-channel scales)
  |
  v
Output (FP32, shape: [1, 2, 6, 6])
```

**Reference**: [AMD Quark](https://github.com/amd/quark)

## ONNX Operations Used

Both models demonstrate the use of:

1. **QuantizeLinear**: Converts FP32 tensors to FP8 format
   - Inputs: `x` (data), `y_scale` (scaling factor)
   - Output: quantized tensor in FP8 format
   - Formula: `y = saturate(x / y_scale)`

2. **DequantizeLinear**: Converts FP8 tensors back to FP32
   - Inputs: `x` (quantized data), `x_scale` (scaling factor)
   - Output: dequantized tensor in FP32 format
   - Formula: `y = x * x_scale`

## Usage

### Generating Models

To regenerate these models, run:

```bash
# Generate NVIDIA ModelOpt style model
python3 generate_nvidia_modelopt_fp8.py

# Generate AMD Quark style model
python3 generate_amd_quark_fp8.py
```

### Converting to Prototxt Format

To convert these binary ONNX models to human-readable prototxt format:

```bash
python3 ../../onnx_prototxt_converter.py nvidia_modelopt_fp8_e4m3.onnx nvidia_modelopt_fp8_e4m3.prototxt
python3 ../../onnx_prototxt_converter.py amd_quark_fp8_e5m2.onnx amd_quark_fp8_e5m2.prototxt
```

### Loading with OpenVINO

```python
import openvino as ov

# Load the model
core = ov.Core()
model = core.read_model("nvidia_modelopt_fp8_e4m3.onnx")

# Compile for inference
compiled_model = core.compile_model(model, "CPU")
```

## Technical Details

### ONNX FP8 Data Types

| ONNX Type | Type ID | Bits | Exponent | Mantissa | Description |
|-----------|---------|------|----------|----------|-------------|
| FLOAT8E4M3FN | 17 | 8 | 4 | 3 | IEEE-like FP8 with wider range |
| FLOAT8E4M3FNUZ | 18 | 8 | 4 | 3 | FP8 E4M3 with no negative zero |
| FLOAT8E5M2 | 19 | 8 | 5 | 2 | IEEE-like FP8 with higher precision |
| FLOAT8E5M2FNUZ | 20 | 8 | 5 | 2 | FP8 E5M2 with no negative zero |

### Quantization Strategies

**NVIDIA ModelOpt** typically:
- Prioritizes E4M3 for better hardware utilization on NVIDIA GPUs
- Uses per-tensor quantization for simplicity and speed
- Optimizes for TensorRT and TensorRT-LLM deployment
- Supports both Post-Training Quantization (PTQ) and Quantization-Aware Training (QAT)

**AMD Quark** typically:
- Supports multiple FP8 formats (E4M3, E5M2) for flexibility
- Uses per-channel quantization for improved accuracy
- Provides cross-platform deployment (AMD, NVIDIA, CPU)
- Includes advanced calibration methods (MinMax, Percentile, MSE, Entropy)
